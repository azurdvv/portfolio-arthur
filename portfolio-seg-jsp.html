Informations sur le

  • Client : Syndicat des Eaux de Gévaudan (SEG)
  • Durée : 4 sprints (6 semaines)
  • Technos : Jakarta EE, Servlets, JSP, Hibernate, Maven, SQL Server
  • Outils : GlassFish, Git/Bitbucket, Jira (Scrum)

Contexte

Le SEG assure le captage, le traitement et la distribution de l’eau potable pour plusieurs communes en régie intercommunale. L’enjeu était de sécuriser l’émission des relevés et factures d’eau pour les usagers.

Dans le cadre de ma formation, j’ai conçu cette application Java Web pour modéliser les relevés, gérer les compteurs et générer automatiquement les rapports de consommation.

Missions

Mission 1 : Mise en place des règles métiers et des tests unitaires (JUnit) pour chaque méthode de la librairie.

Mission 2 : Intégration des annotations JPA pour mapper les entités aux tables SQL Server.

Mission 3 : Développement du tableau de relevés – navigation par commune, secteur et vanne, et affichage des anomalies.

Mission 4 : Création d’un formulaire d’ajout de compteur “Vanne” avec opérations CRUD complètes.
